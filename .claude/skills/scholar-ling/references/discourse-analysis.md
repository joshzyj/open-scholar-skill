# Discourse Analysis and Conversation Analysis Reference

## Conversation Analysis (CA) — Full Notation System

### Jefferson Transcription Notation

```
Turn-taking:
  [            Overlap begins: both speakers talking simultaneously
  ]            Overlap ends
  =            Latching: no gap between end of one turn and start of next

Pauses and silence:
  (.)          Micro-pause (< 0.2 seconds)
  (0.5)        Pause in tenths of seconds
  (( ))        Transcriber description of non-verbal action

Delivery:
  ↑ ↓          Marked pitch rise / fall (on the syllable following)
  °word°       Quieter than surrounding talk (degree signs)
  WORD         Louder than surrounding talk (capitals)
  >word<       Faster than surrounding talk (inward-facing arrows)
  <word>       Slower than surrounding talk (outward-facing arrows)
  wor-         Abrupt cut-off (dash)
  wor:d        Sound elongation (colon; more colons = more elongation)
  .hh          Inbreath (h's)
  hh           Outbreath / laughter (h's without dot)
  £word£       Smiley voice (pound signs)

Transcription uncertainty:
  (word)       Likely transcription but uncertain
  ( )          Indecipherable; one blank per syllable
  ((inaudible)) Inaudible segment

Prosodic boundary:
  ,            Continuing intonation (slight rise)
  .            Falling intonation (strong fall)
  ?            Rising intonation (not necessarily a question)
```

### CA Transcript Example

```
01  Anna: I was thinking we could (0.3) go to the
02         conference on Friday?
03  (0.8)
04  Ben:  Oh:: (.) Friday's a bit (.) tricky for me =
05  Anna:                                    [Mmhm]
06  Ben:  = 'cause I've got this- (0.4)
07         this thing with my department
08  Anna: Right. (.) Okay that makes sense.
09         What about Thursday then?
```

### CA Analysis Procedure

1. **Collect recordings** of naturally occurring interaction (not researcher-designed tasks)
2. **Transcribe** using Jefferson notation; transcription is analytic (you find things while transcribing)
3. **Build a collection** of instances of the target phenomenon across different interactions
4. **Analyze each instance**: What sequential position does it occupy? What comes before and after? What does the next speaker do with it? What interactional work does it accomplish?
5. **Identify the pattern**: What is the practice? What social action does it achieve? What are variants?
6. **Deviant cases**: Find instances that don't fit the pattern; analyze why — often these reveal the rule by showing what happens when it's violated

## Interactional Sociolinguistics (IS)

### Key Analytical Tools

**Contextualization cues**: Linguistic and paralinguistic features that signal "how to interpret this utterance":
- Prosodic: pitch, volume, tempo, rhythm
- Phonological: code-switching, phonetic style shifting
- Lexical/syntactic: formulaic phrases, politeness markers
- Non-verbal: gaze, gesture, posture

**Analysis procedure**:
1. Identify a moment of interactional trouble, mis-alignment, or successful accomplishment
2. Identify the contextualization cues the speaker used
3. Analyze how the interlocutor interpreted/responded
4. Check: Could a different interpretation have been possible? What cues would signal that?

**Footing analysis** (Goffman 1981):
- Principal: the person whose views are expressed
- Author: the person who wrote the words
- Animator: the person speaking the words
- "Participant framework": who has what relationship to the talk

## Critical Discourse Analysis (CDA) — Key Frameworks

### Fairclough's Three-Dimensional Framework

**Level 1: Text** (linguistic analysis)
- Vocabulary: which words are chosen? What connotations?
- Grammar: passive vs. active voice; nominalization (hiding agency)
- Cohesion: how are sentences linked?
- Text structure: what is foregrounded; what is backgrounded?

**Level 2: Discourse practice** (how text is produced and consumed)
- What genre conventions are being invoked?
- What intertextual links exist to other texts?
- What assumptions is the reader expected to share?

**Level 3: Social practice** (social context)
- What social structures and power relations does this text reflect/reproduce?
- What ideological work does it do?

### Van Dijk's Ideological Square

Groups represent themselves as positive and the outgroup as negative through:
- **Positive self-presentation**: emphasize our good deeds; de-emphasize our failures
- **Negative other-presentation**: emphasize their bad deeds; de-emphasize their positive actions

Linguistic strategies:
- **Lexical choice**: "terrorists" vs. "freedom fighters"; "undocumented" vs. "illegal"
- **Presupposition**: embedding ideological claims as given
- **Topicalization/backgrounding**: what is put in subject position; what is in subordinate clauses
- **Evidentials**: marking uncertainty about outgroup claims; certainty about ingroup
- **Number and statistics**: precise statistics for positive facts; vague for negative

### Rhetorical Analysis (Topoi)

Topoi are common argumentative structures in political discourse (Reisigl & Wodak 2001):

| Topos | Structure | Example |
|-------|-----------|---------|
| Usefulness | X is useful/harmful, therefore do/don't do X | "Immigrants contribute to the economy" |
| Danger/threat | X threatens Y, therefore prevent X | "Immigration threatens national identity" |
| Humanitarianism | We have obligations to human beings | "Refugees deserve protection" |
| History | In the past, X happened; therefore now... | "Our country was built by immigrants" |
| Culture | X is part of our culture | "This language is central to who we are" |
| Numbers | There are too many/too few X | "The country cannot absorb more arrivals" |
| Law | X is legal/illegal, therefore... | "We must follow the rule of law" |
| Burden | X costs too much | "Immigrants are a burden on services" |

## Narrative Analysis

### Labov-Waletzky Narrative Structure (1967)

| Component | Function | Identifying question |
|-----------|----------|---------------------|
| **Abstract** | Summary / "What is this story about?" | What was it about? |
| **Orientation** | Sets time, place, persons | Who/what/when/where? |
| **Complicating action** | The narrative events themselves | Then what happened? |
| **Resolution** | How it ended | What finally happened? |
| **Coda** | Bridges past events to present | So that's why... |
| **Evaluation** | The narrator's stance on the events | Why does this matter? What does it mean? |

**Evaluation devices** (how narrators convey significance):
- External evaluation: narrator stops and comments directly
- Suspended action: slowing down to highlight a moment
- Reported speech: direct quotation for dramatic effect
- Comparators: "I could have...", "I might have..."
- Negatives: "I didn't cry" — implies it was cry-worthy
- Intensifiers: "very," "so," "really"; repetition

### Analysis Questions
- What does the narrator choose to tell? (Selection)
- How is the story sequenced? (Does it follow chronology or not?)
- Where is evaluation concentrated? (What was significant to the narrator?)
- What is silenced / not told? (What is conspicuously absent?)
- How does the narrator position themselves vs. others?
- What cultural schemas / ideologies structure the narrative?

## Reporting Qualitative Linguistic Data

### Standards for Presenting Interactional Data
```
All examples should include:
1. Sequential context (at least 2–3 turns before and after)
2. Speaker identifiers (anonymized)
3. Date/setting of recording
4. Minimal but sufficient transcription detail for the argument

Example format:
(2) Excerpt from employment interview, 2022-03-15
    [Job applicant (JA) and interviewer (IR)]

01  IR: Can you tell me a bit about your (.) experience
02      with customer service?
03  JA: Sure (0.3) so I worked at (0.8)
04       [(small laugh)] at this coffee shop for
05  IR: [Mmhm]
06  JA: for about two years and I was
07      basically running the morning shift
```

### Number of Examples to Present

| Purpose | Recommended examples |
|---------|---------------------|
| Central phenomenon | 3–5 instances (enough to show it's a pattern) |
| Deviant/negative cases | 1–2 (show that violations are meaningful) |
| Variation within phenomenon | 2–3 (show range of forms) |
| Statistical tables support qualitative claims | Reference N for collection size |

```
→ points to a specific line being analyzed
```

## Computational CDA and Large-Scale Discourse Analysis

Use when analyzing thousands–millions of texts for discourse patterns. Combines corpus-linguistic precision with CDA interpretive depth.

### Workflow: Corpus-Assisted Discourse Studies (CADS)

1. Build / sample corpus (see `socioling-methods.md` corpus section)
2. Keyness analysis → identify over/under-represented lexis in target vs. reference
3. KWIC → read concordance lines for each key term; identify recurring patterns
4. Collocation analysis → map semantic prosody and association networks
5. CDA close reading → select 20–50 representative texts for detailed analysis
6. Interpretation: connect corpus patterns to Van Dijk / Fairclough frameworks

### Framing Analysis with STM (Structural Topic Models)

```r
library(stm); library(quanteda)

processed <- textProcessor(df$text, metadata = df)
prep      <- prepDocuments(processed$documents, processed$vocab, processed$meta,
                           lower.thresh = 5)

# Fit: K topics; prevalence varies by group and year
stm_fit <- stm(prep$documents, prep$vocab, K = 15,
               prevalence = ~ group + s(year),
               data = prep$meta, init.type = "Spectral", seed = 42)

# Inspect
labelTopics(stm_fit, n = 10)

# Plot highest-probability and FREX words
plot(stm_fit, type = "summary", n = 5, main = "Topic Summary (FREX + Prob)")

# Prevalence effects: does group predict topic use?
effects <- estimateEffect(~ group + s(year), stm_fit, meta = prep$meta)
plot(effects, "group", method = "difference",
     cov.value1 = "right", cov.value2 = "left",
     topics = 1:15, main = "Topic prevalence: right vs. left press")
ggsave("${OUTPUT_ROOT}/ling/figures/fig-stm-frames.pdf", device = cairo_pdf,
       width = 8, height = 5)
```

**CDA interpretation of STM output**:
- Each topic = a potential "frame" or "discourse strand"
- FREX words (high frequency + high exclusivity) = topic-defining vocabulary
- Prevalence effects = which social groups or time periods deploy which frames
- Always follow with KWIC and close reading of representative documents

### LLM-Assisted CDA Coding

Use when manually applying CDA categories (Van Dijk's ideological strategies, Reisigl-Wodak topoi) at scale.

```python
import anthropic, json, pandas as pd
from tqdm import tqdm

client = anthropic.Anthropic()

TOPOI_CODEBOOK = """
Apply the Reisigl-Wodak (2001) topoi framework to identify argumentative strategies.
For each text excerpt, identify which topoi are present (select all that apply):
  - danger_threat: X threatens Y, therefore prevent X
  - usefulness_advantage: X is useful/beneficial, therefore do X
  - humanitarianism: We have obligations to human beings
  - history: The past justifies the present action
  - culture_identity: X threatens our cultural identity
  - numbers: There are too many / too few X
  - law_legality: X is legal/illegal, therefore...
  - burden: X is an economic or social burden

Also identify:
  - positive_self_pres: text emphasizes ingroup positive traits
  - negative_other_pres: text emphasizes outgroup negative traits
  - nominalization: agency is hidden via nominalizations (e.g., "the influx" vs. "migrants arriving")

Return JSON: {"topoi": [list of present topoi], "psp": bool, "nop": bool,
              "nominalization": bool, "key_phrase": "most ideologically loaded phrase"}
"""

def code_cda_batch(texts: list[str], batch_size: int = 10) -> list[dict]:
    results = []
    for i in tqdm(range(0, len(texts), batch_size)):
        batch = texts[i : i + batch_size]
        prompt = TOPOI_CODEBOOK + "\n\nCode these excerpts:\n"
        for j, t in enumerate(batch):
            prompt += f"\n[{j+1}] {t[:500]}"  # truncate very long texts
        msg = client.messages.create(
            model="claude-haiku-4-5-20251001",
            max_tokens=2048, temperature=0,
            messages=[{"role": "user", "content": prompt}])
        raw = msg.content[0].text.strip()
        try:
            parsed = json.loads(raw) if raw.startswith("[") else \
                     [json.loads(l) for l in raw.splitlines() if l.strip().startswith("{")]
        except Exception:
            parsed = [{"topoi": [], "error": "parse_failed"}] * len(batch)
        results.extend(parsed)
    return results

df["cda_codes"] = code_cda_batch(df["text"].tolist())
df["topoi"]     = df["cda_codes"].apply(lambda x: x.get("topoi", []))

# Frequency of each topos
topos_list = ["danger_threat","usefulness_advantage","humanitarianism",
              "history","culture_identity","numbers","law_legality","burden"]
for topos in topos_list:
    df[f"t_{topos}"] = df["topoi"].apply(lambda ts: int(topos in ts))

# Compare frequency across groups
df.groupby("group")[[f"t_{t}" for t in topos_list]].mean().round(3)
df.to_csv("${OUTPUT_ROOT}/ling/tables/cda-topoi-coded.csv", index=False)
```

**Validation** (required for publication):
- Human-code 100–200 excerpts using same codebook → compute κ per topos
- Target: κ ≥ 0.70 per topos category; aggregate κ reported in Methods
- Report inter-run reliability: run same 50 texts twice → compare output

### Text Reuse and Intertextuality

Detect shared language across documents (e.g., legislative boilerplate, media framing templates).

```r
# install.packages("textreuse")
library(textreuse)

# Build corpus
corp_tr <- TextReuseCorpus(dir = "texts/",
                            tokenizer = tokenize_ngrams, n = 5,
                            progress = FALSE)

# Pairwise Jaccard similarity
comparisons <- pairwise_compare(corp_tr, jaccard_similarity)
# Top matches
pairwise_candidates(comparisons, threshold = 0.1)

# Locality Sensitive Hashing (faster for large N)
buckets    <- minhash_generator(n = 200, seed = 42)
lsh_corp   <- TextReuseCorpus(dir = "texts/",
                               minhash_func = buckets, progress = FALSE)
candidates <- lsh_candidates(lsh_corp, threshold = 0.1)
candidates_jaccard <- lsh_compare(candidates, lsh_corp, jaccard_similarity)
```

**CDA use cases**: Detect template spreading in news coverage of immigration; track policy boilerplate across legislative texts; identify cross-platform coordinated framing.
